# 04 — Website Spec

## Sitemap

**Single-page** (all sections live on `index.html`, anchor-linked from a sticky nav). This is intentional — small sites index better and are easier to maintain.

```
/                     → index.html
  #hero
  #about
  #services
  #listings           (OLX iframe + fallback)
  #contact
/robots.txt
/sitemap.xml
/.nojekyll
```

If the owner later needs multi-page (e.g. individual car detail pages or a blog for SEO long-tail), we revisit. Not in scope now.

## Sections — purpose and content

### 1. Nav (sticky)

- Logo (left) — inline SVG wordmark
- Anchor links: About · Services · Listings · Contact
- Language toggle (right): `RO | EN`
- Mobile: hamburger → drawer

### 2. Hero `#hero`

- H1: business name + tagline (bilingual via `data-i18n`)
- One sentence of value-prop
- Primary CTA: **WhatsApp** (`wa.me/<num>?text=<prefilled message in active language>`)
- Secondary CTA: **See listings on OLX** (anchor to `#listings`, which in turn links out)
- Background: subtle automotive photo or solid brand color (decide at design time based on whether owner provides a photo)

### 3. About `#about`

- 2–3 short paragraphs on who IST Auto is, how long they've been operating, what makes them a safe choice
- Optional: small stats row (e.g. years in business, cars sold, etc.) if owner provides numbers

### 4. Services `#services`

- Grid of 3–6 service cards, each with icon + title + short description
- Content pulled from [02-client-data.md](02-client-data.md) Services section (pending owner confirmation)

### 5. Listings `#listings`

- Heading: "Mașini disponibile" / "Available cars"
- **Attempt**: iframe embedding the OLX seller page
- **Fallback** (triggered by `olx-embed.js` if iframe fails to load or returns a same-origin block): large card with "View all listings on OLX →" button linking to `TODO_OLX_SELLER_URL` (target `_blank`, `rel="noopener"`)
- Small disclaimer: "Inventarul este actualizat pe OLX / Inventory is maintained on OLX"

### 6. Contact `#contact`

- Two-column on desktop, stacked on mobile:
  - **Left**: address, opening hours, phone (`tel:`), social buttons (WhatsApp, Facebook, Instagram, OLX)
  - **Right**: embedded Google Map (via `<iframe>` — Maps allows embedding)

### 7. Footer

- Copyright line with current year (generated by JS so it self-updates)
- Small link row: social icons repeated
- `<small>` line: "Site realizat în [year]. Informațiile despre mașini sunt gestionate pe OLX."

## Bilingual approach

**Default language**: Romanian. `<html lang="ro">`.

**Mechanism**: every user-visible text node has `data-i18n="section.key"`. Example:

```html
<h1 data-i18n="hero.title">IST Auto</h1>
<p data-i18n="hero.tagline">Mașini verificate. Livrare în toată țara.</p>
<a data-i18n="hero.cta_whatsapp" href="..."></a>
```

`assets/lang/ro.json` and `en.json`:

```json
{
  "hero": {
    "title": "IST Auto",
    "tagline": "Mașini verificate. Livrare în toată țara.",
    "cta_whatsapp": "Scrie-ne pe WhatsApp"
  }
}
```

`assets/js/i18n.js` responsibilities:

1. Read saved language from `localStorage.getItem('lang')`, default to `ro`
2. Fetch the right JSON, walk `[data-i18n]`, set `textContent` (or `aria-label` / `placeholder` / `title` via `data-i18n-attr="aria-label"`)
3. Update `<html lang>` to match
4. Update the WhatsApp CTA's `href` query param with the language-appropriate prefilled message
5. Toggle listener: click RO/EN switcher → writes `localStorage`, re-runs the walk

**Attributes to translate besides textContent**:

- `data-i18n-attr="placeholder"` for inputs
- `data-i18n-attr="aria-label"` for icon buttons
- `data-i18n-attr="title"` for tooltips

Must test: Romanian diacritics (ă â î ș ț) render correctly after toggle.

## SEO targets

### Per-language meta (rendered into `<head>` by the i18n step, or swapped on language change)

- `<title>` — business name + primary keyword + city, e.g. `IST Auto — Parc auto în <city>, mașini rulate verificate`
- `<meta name="description">` — 150–160 chars, contains primary keywords
- `<meta name="keywords">` — legacy but harmless, include RO+EN terms
- `<link rel="canonical" href="...">`
- `<link rel="alternate" hreflang="ro" href="...">` and `hreflang="en"` + `hreflang="x-default"` → ro
- Open Graph: `og:title`, `og:description`, `og:image`, `og:url`, `og:type=website`, `og:locale=ro_RO` / `en_US`
- Twitter card: `summary_large_image`

### JSON-LD (in `<head>`, single block)

```json
{
  "@context": "https://schema.org",
  "@type": ["AutoDealer", "LocalBusiness"],
  "name": "IST Auto",
  "url": "TODO_SITE_URL",
  "image": "TODO_SITE_URL/assets/img/og.jpg",
  "telephone": "TODO_PHONE_E164",
  "address": { "@type": "PostalAddress", "streetAddress": "TODO_ADDRESS", "addressLocality": "TODO_CITY", "addressRegion": "TODO_COUNTY", "postalCode": "TODO_POSTAL_CODE", "addressCountry": "RO" },
  "geo": { "@type": "GeoCoordinates", "latitude": "TODO_LAT", "longitude": "TODO_LNG" },
  "openingHoursSpecification": [ /* TODO_HOURS */ ],
  "sameAs": [ "TODO_FACEBOOK_URL", "TODO_INSTAGRAM_URL", "TODO_OLX_SELLER_URL" ]
}
```

### Other SEO files

- `robots.txt` — allow all, point to sitemap
- `sitemap.xml` — single URL with `<xhtml:link rel="alternate" hreflang="...">` entries for RO/EN

### Lighthouse targets

- Performance ≥ 90, Accessibility ≥ 90, Best Practices ≥ 90, SEO ≥ 95
- First Contentful Paint < 1.5s on 3G simulation
- Total page weight < 500 KB (excluding iframe content) — enforceable by keeping hero image optimized
